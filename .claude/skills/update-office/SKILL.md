# /update-office — безопасное обновление офиса (v3, pull-only)

Владелец хочет свежую версию офиса. Твоя работа — **обновить константу без потери ни одного
его файла**. Офис v3 по дефолту без push-remote; обновления тянутся из `upstream` (наш публичный
шаблон, только чтение). Если upstream не настроен — подскажи владельцу добавить его один раз.

## Главный принцип

Данные владельца неприкосновенны. Всё, что он наработал, — сначала бэкап, потом обновление
константы, потом возврат его данных поверх. Сбой → **атомарный откат**, офис не остаётся в
промежуточном состоянии.

## Тон

Живо, без терминов. НЕ говори: git, reset, stash, merge, conflict, remote, upstream, commit,
pull. Внутри делаешь это через Bash, владельцу показываешь только человеческий результат.

## Что константа (обновляем), что владельца (бэкап + возврат)

**Константа [T] — приезжает из шаблона, перезаписывается:**
`.claude/hooks/` · `.claude/skills/` · `.claude/settings.json` · `.gitignore` · `.gitattributes` ·
`team/agents/nikita/core.md` + `team/agents/nikita/knowledge/` · заглушки/ядра-шаблоны агентов ·
корневой `CLAUDE.md` (каркас-часть) · `team/agents/_card-template.md`.

**Данные владельца [U] — бэкапим и возвращаем поверх:**
`me/` · `work/` · `clients/` · `projects/` · `results/` · `knowledge/` (summary + raw) · `inbox/` ·
`team/ops/` (build-log, threads, today, ingest-state) · `team/map.md` · `MAP.md` ·
каждый `team/agents/*/memory.md` и `card.md` · сгенерированный характер агентов (Атласа и др. —
он живёт в `card.md`/`memory`, НЕ в core.md, чтобы обновление шаблона его не затёрло) ·
маркер `Ядро собрано: да` и профиль в `me/profile.md` · блок «## Кто владелец офиса» в корневом `CLAUDE.md`.

⚠️ Корневой `CLAUDE.md` — гибрид: каркас-часть [T] + блок ядра [U]. Обновляя, сохрани блок ядра
владельца (между маркером «## Кто владелец офиса» и следующим `##`), наложи на свежий каркас.

## Порядок (атомарно)

1. **Детект платформы и upstream.** `git remote get-url upstream` — нет → предложи владельцу:
   «дай ссылку на свежую версию офиса, я подключу источник обновлений» → `git remote add upstream <url>`.
2. **Бэкап** всех [U]-путей в `/tmp/office-backup-<ts>/` (fallback `$HOME/.office-backup/<ts>/`).
   Проверь, что бэкап реально создан (список файлов), прежде чем трогать что-либо.
3. **Точка отката:** запомни текущий коммит (`git rev-parse HEAD`).
4. **Подтяни константу:** `git fetch upstream` → влей [T]-пути (checkout/merge только константных
   путей; НЕ трогай [U]). Конфликт по [U] невозможен — их не трогаем.
5. **Верни данные владельца** поверх из бэкапа (перезапиши [U]-пути из `/tmp/office-backup-<ts>/`).
6. **Сверка:** пройди по списку [U] — каждый файл на месте и не пустой. Хоть один пропал →
   **откат** к точке из шага 3 + верни бэкап + честно скажи владельцу «обновление отменил,
   ничего не потеряно, разберусь».
7. **Отчёт по-человечески:** что нового приехало (пара строк), данные все на месте.

## Кросс-платформенность

`sha256sum` (Linux) vs `shasum -a 256` (macOS) vs certutil (Windows) — детекть
`command -v sha256sum || command -v shasum`. Windows → Git Bash / WSL. `/tmp` недоступен →
fallback `$HOME/.office-backup/`. Всё остальное (`cp -R`, `find`, `grep`) — POSIX, работает везде.
