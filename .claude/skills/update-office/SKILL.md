# /update-office — безопасное обновление офиса (v3, pull-only)

Владелец хочет свежую версию офиса. Твоя работа — **обновить константу без потери ни одного
его файла**. Офис v3 по дефолту без push-remote; обновления тянутся из `upstream` (наш публичный
шаблон, только чтение). Если upstream не настроен — подскажи владельцу добавить его один раз.

## Главный принцип

Данные владельца неприкосновенны. Всё, что он наработал, — сначала бэкап, потом обновление
константы, потом возврат его данных поверх. Сбой → **атомарный откат**, офис не остаётся в
промежуточном состоянии.

## Тон

Живо, без терминов. НЕ говори: git, reset, stash, merge, conflict, remote, upstream, commit,
pull. Внутри делаешь это через Bash, владельцу показываешь только человеческий результат.

## Что константа (обновляем), что владельца (бэкап + возврат)

**Константа [T] — приезжает из шаблона, перезаписывается:**
`.claude/hooks/` · `.claude/skills/` · `.claude/settings.json` · `.gitignore` · `.gitattributes` ·
`team/agents/nikita/core.md` + `team/agents/nikita/knowledge/` · заглушки/ядра-шаблоны агентов ·
корневой `CLAUDE.md` (каркас-часть) · **все формы шаблона** (basename `_*-template*` + папки
`_template/` и `_memory-template/`): `team/agents/_card-template.md` ·
`team/agents/_memory-template.md` · `team/agents/atlas/_memory-template/` ·
`me/_profile-template.md` · `knowledge/_summary-template.md` · `work/_map-template.md` ·
`clients/_client-template.md` · `projects/_project-template.md` · `team/ops/_today-template.md` ·
`results/_template/`. Формы лежат внутри owner-папок, но владельцу НЕ принадлежат: бэкап их
пропускает (см. exclude в `update-office-sync.sh backup`) — обновлять их обязан ЭТОТ шаг,
иначе форма замрёт на версии дня установки.

**Данные владельца [U] — бэкапим и возвращаем поверх.** ⚠️ **Канонический список путей — функция
`u_paths()` в `.claude/hooks/update-office-sync.sh`** (её же гоняют тесты). Здесь — человеческое
резюме, оно НЕ источник правды; добавляешь новый owner-путь — правь скрипт, не этот абзац:
`me/` · `work/` · `clients/` · `projects/` · `results/` · `knowledge/` (summary + raw) · `inbox/` ·
`team/ops/` (журнал стройки, нити, состояние) · `team/map.md` · `MAP.md` · у каждого агента
`card.md`, `memory.md` и папка `memory/` · блок «## Кто владелец офиса» в корневом `CLAUDE.md`
(извлекается отдельным файлом `owner-block.md`).
Характер и суть агентов живут в `card.md`, НЕ в `core.md` — поэтому обновление шаблона их не трёт.

⚠️ Корневой `CLAUDE.md` — гибрид: каркас-часть [T] + блок ядра [U] (между маркером
«## Кто владелец офиса» и следующим `##`). Слияние делает помощник (ниже), не ручной sed.

⚠️ `card.md` агентов — owner-файлы целиком: три слота сути (Вектор / Помогаю с / Каким видишь)
и строка-трейл рождения возвращаются как есть. `settings.local.json` (накопленные права) не
трогаем вообще: он не константа и не в бэкапе — просто не касайся.

## Механика — через помощник (не руками)

Файловую механику исполняет `.claude/hooks/update-office-sync.sh` — тот же контракт, что гоняют
тесты шаблона. Команды: `backup <dir>` · `restore <dir>` · `owner-restore <dir>` · `verify <dir>`.
Не пересобирай эти шаги вручную из головы — зови помощника.

## Порядок (атомарно)

1. **Детект платформы и upstream.** `git remote get-url upstream` — нет → объясни владельцу:
   добавлять внешние адреса офису запрещено намеренно (защита данных — deny в settings), поэтому
   источник обновлений подключает ОН, один раз, своими руками. Дай готовую строку для его
   терминала: `git remote add upstream <ссылка на шаблон>` — и продолжай после подключения.
2. **Бэкап:** `bash .claude/hooks/update-office-sync.sh backup "$HOME/.office-backup/<ts>"`.
   **Только домашняя папка владельца, НИКОГДА `/tmp`:** в бэкапе лежат карточки клиентов, а `/tmp`
   на маке/линуксе читается любым пользователем и процессом машины — это утечка ПД мимо всех гейтов.
   Помощник сам ставит папке права 700 и откажется писать в непустой каталог. Он сложит [U]-пути +
   owner-блок отдельным файлом `owner-block.md` (страховка вне зоны правки). Проверь вывод
   `backup=ok files=N`; N=0 или ошибка → СТОП, ничего не обновляй.
3. **Точка отката:** запомни текущий коммит (`git rev-parse HEAD`).
4. **Подтяни константу:** `git fetch upstream` → влей [T]-пути (checkout/merge только константных
   путей; НЕ трогай [U]). Конфликт по [U] невозможен — их не трогаем.
5. **Верни данные владельца:** `… update-office-sync.sh restore <dir>`. Помощник вернёт [U]-пути
   и вошьёт owner-блок в свежий CLAUDE.md. Смотри его вывод:
   - `owner=ok` — блок на месте, едем дальше;
   - `owner=lost` — маркер-секция в CLAUDE.md потеряна (человек мог снести заголовок). НЕ падай
     и НЕ стопай обновление: это self-serve случай. Покажи владельцу найденный блок из бэкапа
     («вот что я знаю о тебе — вернуть в главный файл?») и по «да» выполни
     `… update-office-sync.sh owner-restore <dir>`. Жёсткий стоп — только если владелец сам
     отказался или блок в бэкапе пуст.
6. **Сверка:** `… update-office-sync.sh verify <dir>` — каждый файл владельца на месте и не
   пустой. `verify=fail` → **откат** к точке из шага 3 + снова `restore` из бэкапа + честно
   скажи владельцу «обновление отменил, ничего не потеряно, разберусь».
7. **Убери бэкап:** `… update-office-sync.sh cleanup <dir>` — ТОЛЬКО после `verify=ok`. Копия ПД
   не должна жить на диске дольше обновления. `verify=fail` → бэкап НЕ трогай, он ещё нужен.
8. **Миграция вектора (одноразовая, для офисов старше этой версии).** Есть `me/vector.md`, а слот
   «Вектор» в `team/agents/atlas/card.md` пуст → перенеси цель ДОСЛОВНО в слот, файл сдвинь в
   архив и скажи владельцу одной строкой: «твою цель перенёс в карточку коуча — он теперь держит
   её при себе». Слот уже заполнен → ничего не делай, `me/vector.md` просто в архив.
9. **Отчёт по-человечески:** что нового приехало (пара строк), данные все на месте.

## Кросс-платформенность

`sha256sum` (Linux) vs `shasum -a 256` (macOS) vs certutil (Windows) — детекть
`command -v sha256sum || command -v shasum`. Windows → Git Bash / WSL. `/tmp` недоступен →
fallback `$HOME/.office-backup/`. Всё остальное (`cp -R`, `find`, `grep`) — POSIX, работает везде.
