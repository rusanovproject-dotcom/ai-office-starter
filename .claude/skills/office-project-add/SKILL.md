# office-project-add — механика проектов (рефлекс корня №2)

Одна логика навсегда: как заводится ОДИН проект, так заводятся все. Человек говорит
«добавь проект <название>» — ты разворачиваешь его дом.

## Что делаешь

1. **Slug** из названия (латиница-кебаб, коротко): «Запуск курса» → `zapusk-kursa`.
2. **Создай** `projects/<slug>/` с файлом `map.md` внутри — дом проекта: что это, цель,
   статус, что уже сделано. Пустых папок не плодим — сразу с домом.
3. **Строка в карту** `projects/map.md`: `- **<Название>** — <одна фраза> · keywords: <2-4 слова
   ЧЕРЕЗ ЗАПЯТУЮ>` (keywords питают детект проекта в prompt-inject; разделитель — запятая,
   слова через пробел слипнутся в один мёртвый ключ).
4. **Обнови** INDEX и, если создан, MAP.md (правило самопополнения: кто создал — тот вписал).
5. Скажи человеку по-человечески: «Завёл проект «…». Теперь всё по нему — сюда. Скажешь про него —
   офис сам поймёт, что это он.»

## Раскладка клиентов («разбери/разложи клиентов», «наведи порядок в клиентах»)

Механика (формат — `clients/CLAUDE.md`): один человек = одна карточка
`clients/<slug>/README.md` (кто, контакт, история, договорённости) → после каждой карточки —
строка в `clients/INDEX.md`. Большая пачка (таблица, свалка) → частями, с честным отчётом
(«разложил 40 из 300, продолжаю»). Содержимое записей человека не редактируешь — только
раскладываешь.

## ANTI-триггер порядка (рефлекс №7)

Разводи два разных «наведи порядок»:
- **«в клиентах / делах / задачах»** → это ЗДЕСЬ: разложить клиентов по `clients/`, дела по
  проектам, актуализировать карты. Занимаешься содержанием.
- **«в офисе / файлах / папках»** → это ЧИСТКА структуры → скилл `office-tidy` (лицо — Рита
  с этапа 8; до тех пор ведёт Никита, по-честному: «полноценную хранительницу Риту поставим
  на финале стройки»).

Не перепутай: просьбу про клиентов не превращай в уборку служебных папок.
