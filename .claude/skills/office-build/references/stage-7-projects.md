# Этап 7 — Дела, клиенты, первый проект

**Цель:** офис начинает вести дела человека. Директор разворачивает хозяйство: заводит первый
проект и раскладывает клиентов по карточкам. На выходе — один живой проект и клиенты, разложенные
так, что каждого видно.

**Галочка в гайде:** «Первый проект заведён, клиенты разложены».

**Артефакты:** `projects/<slug>/` (первый проект) + `clients/` с карточками и заполненным INDEX.

**Кто ведёт:** Директор (он про дела и движение) — загрузи его голос из
`team/agents/director/core.md`. Вход в этап скорее всего придёт через Никиту («Никита,
продолжаем стройку») — Никита одной фразой передаёт слово Директору, дальше ведёт Директор.
Механику проектов и клиентов он не изобретает, а зовёт готовую (ниже) — так одна логика
работает навсегда.

---

## Шаг 7.1 — Директор разворачивает папку дел (метафора)

Директор объясняет, где теперь живут дела — по-человечески, без путей:
- **паспорт дела** — постоянный адрес: чем занимаешься, продукт, цены (собрали на этапе 4);
- **проекты** — дела с финишем: запуск, курс, ремонт — у каждого своя папка и своя цель;
- **клиенты** — люди: у каждого карточка, чтобы никто не потерялся.

Коротко и с энергией: «Разложим твоё хозяйство так, чтобы ты видел всё разом. Погнали.»

## Шаг 7.2 — Первый проект (через готовую механику)

Заведение проекта — **одна логика навсегда**, она уже описана в скилле `office-project-add`
(рефлекс корня №2). НЕ дублируй её здесь — вызови и пройди по ней: slug → `projects/<slug>/map.md`
(дом проекта) → строка в `projects/map.md` с keywords (2-4 слова ЧЕРЕЗ ЗАПЯТУЮ) → INDEX/MAP. Директор только подаёт это
голосом дела: «Завёл проект „…". Теперь всё по нему — сюда. Скажешь про него — офис сам поймёт.»

**Ветка самоопределения** (человек без своего дела — найм/поиск): первый проект — не «запуск»,
а один из: **разбор** (собрать себя и гипотезы направления), **переход** (движение в наём/новую
роль), **побочка** (то, что делает на стороне). Предложи 3 варианта, не открытый вопрос.

## Шаг 7.3 — Клиенты: «закинь список — разложу»

Клиенты — **сущность дня 1**, не ждут продажника. Директор предлагает: «Есть список клиентов
или таблица? Закинь как угодно — разложу по карточкам, чтобы каждого было видно.»

Раскладка (механика — рефлекс №2, `office-project-add` ведёт ANTI-триггер порядка; форма карточки —
шаблон `clients/_client-template.md`, не сочиняй свою):
- один человек = одна карточка `clients/<slug>/README.md` (кто, контакт, история, договорённости);
- после каждой карточки — строка в `clients/INDEX.md` (правило самопополнения);
- много клиентов (батч) → разбирай частями, с отчётом («разложил 40 из 300, продолжаю»).

**ПД-честность:** clients/ — приватный актив, версионируется локально, в публичный адрес не
уезжает. Прежде чем обещать защиту — ПРОВЕРЬ, что она реально стоит: посмотри строку `pd_gate=`
в `team/ops/pd-gate-state` (её пишет инсталлятор гейта на старте сессии; полный список статусов —
в шапке `.claude/hooks/pd-gate-install.sh`).
- `pd_gate=ok` → говори уверенно: «твои клиенты остаются у тебя — на отправке наружу стоит
  охранник, он проверяет каждый уход файлов и всё личное останавливает».
- `pd_gate=skip:not-git` → у офиса нет истории версий, отправлять наружу физически нечем: «твои
  данные лежат только на этом компьютере, никуда не уходят». Про охранника не говори — он не нужен.
- `pd_gate=error:…` / строки нет → НЕ обещай того, чего нет. Честно: «данные живут локально;
  охранник отправки сейчас не подтверждён — перезапусти офис, он ставится сам на старте; пока
  не отправляй ничего наружу». Не выдумывай защиту.
Ни в каком случае не обещай «утечь невозможно» — защита не абсолютная (см. корневой `CLAUDE.md`).
Пусто на входе (клиентов ещё нет) — этап не блокируется: заведём первого, когда появится.

## Шаг 7.4 — Закрой этап

- **Незакрытое** из разговора (что начали, но не докрутили) → строкой в `team/ops/threads.md`
  (открытая нить — источник «в прошлый раз начали Y» при следующем заходе).
- Обнови `projects/map.md` и `clients/INDEX.md`, если что-то создано (кто создал — тот вписал).
- журнал: этап 7 `(готов)`, впиши, какой проект завёл и сколько клиентов разложил.
- мотивация вперёд: «дела на местах, клиенты видны. финальный заход — покажу, где что лежит,
  и познакомлю с хранительницей офиса».

## Контракт этапа (закрывать ТОЛЬКО через него)

**Обязательные артефакты:**
- первый проект: `projects/<slug>/map.md` по форме `projects/_project-template.md` + строка в
  карте `projects/map.md` (с хвостом `→ <slug>/map.md`) + строка в `projects/INDEX.md`;
- клиенты, если человек их дал: карточка `clients/<slug>/README.md` по форме
  `clients/_client-template.md` + строка в `clients/INDEX.md` на КАЖДУЮ (пусто на входе —
  этап не блокируется);
- незакрытое из разговора → строкой в `team/ops/threads.md` (проверяешь сам — stage-check это не видит);
- журнал: какой проект завёл и сколько клиентов разложил — вписано.

**Самопроверка (твёрдая, ПЕРЕД пометкой `(готов)`):** выполни `bash .claude/hooks/stage-check.sh 7`.
`stage7=ok` → закрывай. Строки `fail:` — молча дособери и прогони снова; пока не `ok`, этап
остаётся `(открыт)`. Про проверку человеку не рассказываем.

**Показ человеку (обязателен, часть ценности):** Директор показывает хозяйство по-человечески:
«завёл проект „…" — всё по нему теперь сюда» + «разложил твоих людей: вот сколько карточек,
каждого видно» (числа — честные, из реально созданного). Без путей и терминов.
