## Как работает этот офис (для помощников)

Офис — команда цифровых сотрудников. У каждого своя папка в `team/agents/<имя>/`
и свой характер (`core.md`). Общая опора всех — **выжимка** `knowledge/summary.md`:
кто владелец, чем занимается, каким голосом говорит. Читаешь её перед содержательной работой.

Память офиса — файлы. Ничего не теряется: сырьё в `knowledge/raw/`, готовое в `results/`,
люди в `clients/`, дела в `work/` и `projects/`. Пустых папок не заводим — что не создано
лестницей стройки, того на диске нет.

**Персоны по событию:** `core.md` агента грузится ТОЛЬКО когда идёт разговор с ним по имени.
Атлас не инжектится каждую сессию. Бюджет always-on ≤400 строк, карта команды ≤5 строк/агент.

**Как работает роутинг (и как он растёт сам).** Задача НЕ всегда идёт через одного «главного» —
хук `prompt-inject.sh` детерминированно направляет тремя проходами: (1) назвал агента по имени →
сразу к нему (из `team/map.md`); (2) горячее намерение → сразу к скиллу (ключи берутся из строки
`route:` в frontmatter каждого скилла; хвостовая `*` в ключе = любые словоформы: `галочк*`); (3) размыто / 2+ совпадения → офис уточняет, к кому, не
угадывает молча. **Источник ключей — сами скиллы, не список в хуке.** Поэтому роутинг
самопополняемый: добавил агента в карту / скилл с `route:` — он сразу маршрутизируется, править
хук не нужно. Нет совпадения — хук молчит, дальше решает модель. Единственный inject-хук — этот.

## Что куда кладём (детерминированный выбор, агент не гадает)

| Тип вещи | Папка |
|----------|-------|
| пост / страница / письмо — то, что владелец ПОКАЗЫВАЕТ | `results/` |
| факт о деле (продукт, цены, позиционирование) | `work/` |
| человек (клиент, лид) | `clients/` |
| файл, который кинул владелец | `inbox/` → после разбора в `knowledge/raw/` |
| дистиллят: кто он, чем занят, каким голосом | `knowledge/summary.md` (выжимка) |
| проект (у дела есть дата завершения) | `projects/<slug>/` |
| нить разговора «в прошлый раз начали Y» | `team/ops/threads.md` (создаётся при первой нити; пиши строкой с колонки 0, самая свежая — сверху; закрыл нить — удали строку) |
| решение/лог сотрудника | его `team/agents/<имя>/memory.md` (у коуча — папка `memory/`) |

**Правило самопополнения:** кто создал файл — тот дописал строку в INDEX (только в папках, где он
есть: `clients/`, `knowledge/`, `projects/`) и обновил `MAP.md`. На чистках это сверяет Рита
(файл без строки в INDEX / строка без файла = дрейф).

## Безопасность (константа, менять нельзя)

- Офис по дефолту **без push-remote** — живёт локально, снапшоты локальные. Обновления шаблона
  тянутся из `upstream` (pull-only). Push настраивается только по явной просьбе владельца.
- `.env`, `*.pem`, `secrets*`, медиа — в git не едут (`.gitignore` + deny в `settings.json`).
- `clients/` версионируется (это актив), но публичный remote + карточки клиентов → pre-push блокирует.
- Медиа-честность: видео/аудио сохраняем в `knowledge/raw/media/`, но говорим правду —
  «сохранил, расшифровывать пока не умею, эта суперсила приедет позже». Не обещаем того, чего нет.
